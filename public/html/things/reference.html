What are OOCSI Things?

Super quick: it's a collection of small mobile web pages that each do one thing well and communicate data via the OOCSI design middleware. Each thing has a different specialty: sensing data, allowing for user interaction, playing media, or all of them.

Think of them like walkie-talkies. When multiple Things join the same "Team Space" (like tuning into the same radio channel), they can instantly hear each other and share information, such as button presses, colors, or sensor readings.

Useful?

OOCSI things are great for prototyping connected interactions without coding, so you should get simple interactions going within minutes. Things run on smart phones, tablets and laptops, and you can indeed prototype with these devices by building small enclosures or fixing them to other building blocks.

How it works

After connecting to OOCSI, a thing will join a team space (actually an OOCSI channel) and then exchange data with other things over this channel. The data exchange uses the OOCSI variable abstraction which allows to synchronize the variable values quickly and safely over the OOCSI channel.

  • Defining variables: You define a variable that will be shared across the network (e.g., let position = oocsiThings.data('position');).
  • Reading values: To get the current value, simply use the variable name with empty parentheses: position().
  • Writing values: To change the variable across all devices, pass the new value into the parentheses: position(2).

Everywhere in the code of the thing, you can use these variables to change the behavior or to share new values with other things.

Installation

Quick start

To work with an OOCSI thing, for instance, to modify it, extend it or change any part of it, click the Download source link that you can find in the starting dialog of every thing (that's where you select the team). This will download a single HTML file that you can then upload to a web-accessible directory of your choice. You can use Data Foundry to do that (just create a new Data Foundry project with an "existing dataset", upload your HTML file there and activate web access).

Things need libraries

We need a few libraries loaded in the head of any thing HTML file: 


<link href="/assets/stylesheets/picoCSS/pico.indigo.min.css" rel="stylesheet" type="text/css">
<script src="/assets/javascripts/oocsi-web.js"></script>
<script src="/assets/javascripts/things/oocsi-things.min.js"></script>
The first is a stylesheet that ensures that navigation, and especially the team selection dialog look right. The second and third are scripts, one for OOCSI and the other for the thing features that build on top of OOCSI.

Some things have more libraries because they need to access sensors, produce sound or draw something on a canvas. We often use P5.js for that. P5.js is a great fit for OOCSI Things because its continuous draw() loop allows you to read OOCSI variables (like toggle()) and update the screen instantly without needing complex event listeners.

Setup a place for thing code

The setup of the thing should happen in one go and before the rest of the page starts rendering. As a convention we use a final function that you define in the beginning of a JavaScript code block in your HTML page. 


<script>
    // define thing variables here, so you can use them everywhere
    let thingsVariable1;

    // this is the entrypoint for your thing
    function thing() {
      // all your thing code goes here
      // ...
    }

    // other functions go here, such as setup() and draw() if you use P5.js
</script>
Note that before we define varaiables outside this function, otherwise we cannot use them in the rest of our code.

Select a team space

By default, OOCSI Things will pop up a dialog for you to select a team space on every load (for every thing). It will store the last selection in a cookie on your device, so you don't need to enter the name of the space again and again. The more you get into prototyping, the less this is needed (and quite annoying). So, you can preset the team space in your things like so: 


<script>
    
    // define your teamspace
    oocsiThingsConfig = {
      // enter your team name, or leave empty for team selection dialog
      teamspace: "team-25"
    };

    function thing() {
      // ...
    }
</script>

This is all we need to do for the setup. Let's see what the different functions inside the thing() function do exactly.

Function reference

OOCSI things functions

oocsiThings.register(...)
Description

This is the first function to call in your thing code. This function registers the thing and connects it to the teamspace.

Parameters
  • name (string): screen name of the thing, can include emjois and will be shown in the header of the thing
  • handle (string): OOCSI name of the thing, only alphanumerical characters and dashes allowed. This will be used on the OOCSI network.
  • inVars (array of string): list of input data variables, just for documentation and showing input indicators in header of the thing.
  • outVars (array of string): list of output data variables, just for documentation and showing input indicators in header of the thing.
Example Usage 

// from the lampo thing
oocsiThings.register('Lampo 🪔', 'lampo', ['toggle', 'dim'], ['toggle'])

Note: If you duplicate a thing, for instance, because you need two or more things with the same functionality, you need to give them different names to ensure that they are not treated same on the OOCSI network. Even if you give them different names, they can share the same data.

oocsiThings.data(...)
Description

The data() function creates the data variables that are synchronized via the OOCSI network.

Parameters
  • name (required): The name of a data variable on the OOCSI network. This name can be different than the variable name in your code. Nevertheless, we suggest to keep them same to not lose overview.
Example Usage

Here is an example of how to use the data() function: 


// we first define the variable "toggle" outside the thing() function
let toggle;

// create the variable
toggle = oocsiThings.data('toggle')

// set the initial value --> true
toggle(true)

Note: Don't forget to define the variable in your code _outside_ the thing() function. Otherwise, you cannot access it in other parts of your JavaScript code.

oocsiThings.link(...)
Description

The link() function links different variables between different Things. This is how you make one Thing control or affect another.

Parameters
  • thing_name (required): this is the handle of the other Thing whose data you would like to receive.
  • data_name (required): this is the name of the data variable on the other Thing.
  • own_data_name (required): this is the name of your local data variable that will receive updates.
Example Usage

Below is an example of how to use the link() function. Imagine you have a slider Thing called touchee (which outputs a variable called y) and a light Thing called lampo (which uses a variable called dim for its brightness):


// In the lampo Thing's code:
// Link the "y" (slider position) data from any "touchee" thing to our local "dim" data
oocsiThings.link('touchee', 'y', 'dim')

Note: The values of linked variables are not converted automatically, please test carefully.

oocsiThings.activity()
Description

The activity() function just shows dat aexchange activity in the header of the thing. This can be useful for debugging your thing or simply to show activity.

Example Usage:

Here is an example of how to use the activity function:


// some data update has happened
// ...
// show activity in user interface
oocsiThings.activity()

Data variable functions

The functions below apply to all data variables that you have created with oocsiThings.data(...) (see above).

subscribe(...)
Description:

The subscribe() function subscribes to changes in a variable and allows to trigger code on every change. Is not specific to the oocsi things library. Instead it belongs to each OOCSI variable.

Parameters
  • function (required): The code to run when a change in the data variable occurs.
Example Usage

Here is an example of how to use the subscribe function: 


// a data variable "toggle"
let toggle;

// linked to the "toggle" data item on the teamspace
toggle = oocsiThings.data('toggle')

// subscribe with a function to run on any data change
toggle.subscribe((newValue) => {
  // do something with the new value
  // e.g., log the value on the console
  console.log(newValue)
})

Note: The code will be run on all variable updates, but only if the value changes. This prevents that you run the code multiple times on a single repeated update.

Getting values <var>()
Description

This function simply returns the current value of the data variable.

Parameters
  • none
Example Usage 

// a data variable "toggle"
let toggle;

// linked to the "toggle" data item on the teamspace
toggle = oocsiThings.data('toggle')

// ...

// read data variable to display something in a data-driven way:
// here we check the data value in an if statement
if(toggle()) {
  // if true, we use a light fill color (in P5.js)
  fill(255);
} else {
  // if false, we use a dark fill color (in P5.js)
  fill(0);
}
Setting values <var>(...)
Description

This function simple sets a new value and the synchronization with other things will be started automatically.

Parameters
  • newValue (required): The new value for this data variable.
Example Usage 

// a data variable "toggle"
let toggle;

// linked to the "toggle" data item on the teamspace
toggle = oocsiThings.data('toggle')

// ...

// in a P5.js touch handler, we change the value of the toggle variable
function touchEnded() {
  // read the current value
  let current = toggle();
  // set the inverted value
  toggle(!current);
}

Communicating beyond OOCSI Things (e.g., ESP32)

Because OOCSI Things run on standard OOCSI channels (the Team Space), any external OOCSI client can communicate with them. This is incredibly powerful for hybrid prototyping where you mix web interfaces with physical sensors and actuators powered by microcontrollers like an ESP32.

Channel naming

To talk to your Things, your external client needs to connect to the exact same Team Space. If your Team Space is called team-25, your ESP32 needs to send messages to, or subscribe to, the channel named team-25. In most settings, this channel might be prefixed, such as OOCSI-things/team-25.

Sending data to Things (ESP32 Example)

If you have a physical button connected to an ESP32 and you want a Thing to react to it, you send a standard OOCSI message to the shared channel: 

// ESP32 Code (using OOCSI-ESP library)
// Create a message for the Team Space channel
oocsi.newMessage("OOCSI-things/team-25");

// Add data. The key "cup_drink" becomes the variable name in the Thing
oocsi.addBool("cup_drink", true);

// Send it!
oocsi.sendMessage();

In your web Thing, you would define a variable (e.g., let drink = oocsiThings.data('drink');) and then link it to the ESP32 (acting as thing "cup") using its handle: oocsiThings.link('cup', 'drink');.

Receiving data from Things (ESP32 Example)

Conversely, if you want an ESP32 to react to a Thing (for example, tapping a button on a web page to move a physical servo motor), the ESP32 subscribes to the channel and listens for the Thing's variable name: 

// ESP32 Code (using OOCSI-ESP library)
// First, subscribe to the channel in your setup()
oocsi.subscribe("OOCSI-things/team-25");

// Inside your processOOCSI callback function:
void processOOCSI() {
  // Check if the message contains the variable from your Thing (e.g., "lampo_toggle")
  if (oocsi.has("lampo_toggle")) {
    // Get the value sent by the Thing
    bool toggle = oocsi.getBool("lampo_toggle", false);

    // Use the value to move a servo motor
    myservo.write(toggle ? 0 : 180);
  }
}
